# Atlas Style · 科研图鉴风格规范

把 `docs/贷款申请客户旅程看板.html` 中打磨并验证过的编辑排版风格固化为可复用规范。
目标观感：像一页科学期刊图版/研究报告——安静、考究、信息密度高、每个元素都有出处。

## 何时读 references/tokens.md

`references/tokens.md` 存有完整的 CSS 变量块、组件样式片段和可直接复制的骨架。
写任何新页面前先读它，把 CSS 变量块原样拷入 `<style>` 开头，再按下面的规则组装。

## 三条不可妥协的底线

1. **配色只用已验证的色板**（见下），不新造颜色。三色组合已通过色觉无障碍校验
   （最差色对 CVD ΔE 20.1）；换色必须重跑 dataviz 技能的 validate_palette 脚本。
2. **文字永远穿文字色**（墨色系），不穿数据色。彩色只出现在标记（色块、弧线、圆点）上，
   身份靠文字旁边的彩色标记传达。例外：琥珀金/砖红可用于强调性关键词，但正文不行。
3. **专业感高于趣味性**。禁止手绘、抖动、涂鸦效果——本仓库用户明确否决过手绘风
   （"太潦草，要保持专业度"）。强调用干净的描边、色彩和粗细，不用"随手画"的模拟。

## 色板（亮色纸面，已验证）

| 角色 | 变量 | 值 | 用途 |
|---|---|---|---|
| 页面底 | `--page` | `#f3efe2` | body 背景（米灰） |
| 图面纸 | `--paper` | `#fbf8ef` | 卡片/图表面 |
| 侧栏 | `--panel` | `#e9eef0` | 速览面板（冷灰蓝） |
| 主墨 | `--ink` | `#24272e` | 标题、正文、数值 |
| 次墨 | `--ink-2` | `#565a63` | 说明文字、次级标注 |
| 弱化 | `--muted` | `#8b8574` | 眉标、轴标（暖灰） |
| 细线 | `--hairline` | `#d9d2bf` | 边框、分隔线、网格 |
| 主色 | `--blue` | `#38699f` | 数据主体（石板蓝） |
| 强调 | `--accent` | `#b5502f` | 摩擦点/异常/警示（砖红） |
| 辅助 | `--ai` | `#e0a000` | 第二维度，如 AI 相关（琥珀金） |

语义纪律：蓝 = 数据主体；红 = 需要注意的东西（稀用，一页只强调一件事）；
琥珀 = 与主体并存的第二维度。三色不混用职责。

## 字体与文字层级

- 衬线（标题/正文/大数字）：`Georgia, "Times New Roman", "Songti SC", "STSong", "SimSun", serif`
- 等宽（眉标/标注/数值列/轴标）：`ui-monospace, "SF Mono", Menlo, Consolas, "Courier New", monospace`
- 页面大标题：衬线 28–30px / 700；卡片标题：衬线 15–17px / 650
- **眉标（eyebrow）**：等宽 10–11px、`letter-spacing: 0.18–0.22em`、大写，
  格式 `<span class="no">0X</span> · SECTION NAME`，编号用 `--accent` 色
- **图注（figcap）**：衬线斜体 12.5px `--ink-2`，以 `**图 N** ·` 开头；每张图/表都要有图注，
  图注承担图例职责（彩色关键词直接写进图注句子里）

## 结构语法

- 全页分节编号 01、02、03……，每节一个眉标；图表编号 图 1、图 2……独立计数
- 卡片：`--paper` 底 + 1px `--hairline` 边 + 3px 圆角（圆角要小，编辑风不是卡片风）
- 标题下用 1.5px `--ink` 实线收口（`header .rule`）；表格首行上方同样用 1.5px 墨线
- 点阵纹理：大图底面铺 26px 间距、r=0.9、`rgba(36,39,46,0.055)` 的点阵网格（SVG pattern）
- 数据面积填充用点阵（stipple：7px 间距、r≈1.15、主色 40% 透明度），不用实色块
- 注释框：白底 + hairline 边 + 细引线（0.8px）连到目标；小装饰（罗盘、入口/终点标记）
  用等宽小字 + `--muted`，克制使用

## 图表纪律（与 dataviz 技能配合）

- 折线 2px 圆头；条形 ≤24px 厚、数据端 4px 圆角、基线端直角；网格 1px 实线 `--hairline`
- Y 轴刻度用 nice-number（步长取 1/2/2.5/5×10ⁿ），绝不出现 163 这类脏刻度
- 编码要诚实：宣称"面积∝数值"就不许加基数或上限压缩（可设最小可读尺寸并在图注声明）
- 悬停提示：白底 `#fffef9` + 1px `--ink` 边 + 硬阴影 `3px 3px 0 rgba(36,39,46,0.12)`，
  数值粗衬线在前、标签次墨在后
- 交互元素要可键盘到达：`aria-label` 写全称，重渲染后归还焦点，focus 与 hover 同权

## 反例（见过就不要再犯）

- ✗ 手绘/抖动圈注 → 用干净的红描边或红色数值强调
- ✗ 大圆角、阴影卡片、渐变 → 这是 SaaS 风不是图鉴风
- ✗ 彩色正文文字、每个数据点都标数值 → 选择性直标 + 悬停 + 表格视图兜底
- ✗ "今日"标签配 T+1 数据 → 口径词要诚实（写"昨日 (T+1)"）
