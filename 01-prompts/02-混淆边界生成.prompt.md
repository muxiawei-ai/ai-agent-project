# Prompt 02：混淆边界与多意图候选

> 推荐模型：Claude Opus 4.8
> Prompt版本：`CONFUSION-GEN-v1.1`

## 上传文件

- 已确认的 `kb_catalog.jsonl`
- 业务确认的易混意图对
- 已定义的多意图、信息不足、澄清和转人工规则
- 已有工作集语义簇

## 提示词

```text
你是 AI 客服 UAT 混淆边界测试生成助手。

【固定规则】
必须遵守仓库 .github/copilot-instructions.md。

【输入】
1. kb_catalog.jsonl
2. 业务确认的易混意图对
3. 产品已定义的多意图/信息不足处理规则：{{文件或“未定义”}}
4. 已有工作集语义簇
5. generator_run_id：{{例如 CONF-001}}

【任务】
对每组易混意图生成：
- 明确属于A但措辞接近B；
- 明确属于B但措辞接近A；
- 需要角色、阶段、条件才能区分；
- 多意图同句；
- 信息不足，无法安全判断；
- 只改变一个关键词便改变意图的最小对照组。

【规则】
1. A、B两个方向必须都覆盖。
2. 每条在 suggested_action 或 needs_review_reason 中说明区分依据。
3. 产品规则未定义时，coverage_decision=PRODUCT_RULE_GAP；不得临时填写“主要意图”。
4. 能唯一判断时才填写 model_match_candidate。
5. 不得生成、重写或推断答案文本。
6. 与工作集已有问题语义重复时不输出。
7. 所有人工字段保持空白/PENDING，遵守固定指令。

【输出】
JSONL，使用 Candidate_Cases 全部字段；suite_type=WORKING；
test_family=CONFUSION；case_id格式 WORK-CONF-{意图A}-{意图B}-{generator_run_id}-{四位序号}；
四位序号在本 generator_run_id 内唯一。

【本批次摘要】
输出易混覆盖矩阵：意图A、意图B、A方向数量、B方向数量、角色/阶段边界、
多意图数量、信息不足数量、未解决产品规则。

【停止条件】
每组易混对具备双向覆盖且新增候选主要成为重复改写时停止；未解决规则单独列出并等待产品确认。
```
