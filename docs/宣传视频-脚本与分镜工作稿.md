# AI 客服项目宣传视频 · 脚本与分镜工作稿（v0.1 草稿）

> 状态：创意草稿，待业务确认核心信息与受众后进入分镜细化。
> 时长目标：90 秒（可裁剪出 60 秒精简版和 30 秒预告版）。
> 素材基础：App 内真实调用、点击、操作的录屏（持续补充中）。

## 一、定位（先回答三个问题，再动手剪）

| 问题 | 当前草案 | 待确认 |
|---|---|---|
| 给谁看？ | 管理层 / 内部各部门（宣传项目价值） | 如果面向客户或外部，第三幕话术需调整 |
| 一句话核心信息 | **「答得准、不乱答」——每一句回复都有出处，每一个风险都有兜底** | 业务确认口径 |
| 看完希望对方记住什么？ | 这不是一个"会聊天的 AI"，而是一套**可控、可验证、可追溯**的客服质量体系 | |

原则：视频里出现的每一句 AI 回复，都必须是知识库里真实批准过的内容——宣传片本身也遵守"一切可追溯"。

## 二、90 秒三幕结构

### 第一幕 · 问题（0–15s）——制造张力，不出现产品

- 黑场起，快节奏文字快闪：真实客户的各种问法（同一个问题的 5 种问法、错别字、口语化）。
- 节奏渐快，叠加一条红色的敏感问题（如收费承诺、法律责任类），音乐骤停。
- 定格字幕：**「客户什么都会问。AI 客服，敢答吗？」**

### 第二幕 · 解决（15–65s）——录屏主战场，三个场景递进

| 场景 | 演示内容 | 传递的点 |
|---|---|---|
| 1. 答得准 | 客户正常提问 → AI 秒回标准答案；镜头推近回复气泡，字幕标注"出自批准知识库" | 口径可控，不是 AI 自由发挥 |
| 2. 听得懂 | 同一问题换三种花样问（口语、错字、绕弯）→ 三次都命中同一条正确答案，三画面并列对比 | 客户怎么问都能找对 |
| 3. 不乱答 | 敏感 / 超范围问题 → AI 礼貌拒答并转人工；画面定住，打出**「宁可不答，不能错答」** | 差异化卖点：敢拍"AI 拒绝回答"的宣传片，本身就是自信 |

场景 3 是全片情绪最高点，音乐和节奏在这里做最大反差（骤停 → 沉稳收住）。

### 第三幕 · 证明与收尾（65–90s）——数字与承诺

- 大字报式数据卡片依次入场（来自项目底线标准）：
  - 敏感与合规红线问题处置正确率 **100%**
  - 超范围问题 **零**强行作答
  - 每一句回复 **可追溯**到批准来源
  - 上线后每月 **1–10 万条**真实数据持续驱动优化
- 收尾镜头：知识库 → 测试 → 上线 → 数据回流的闭环示意动画。
- 定版 slogan + 项目名 + 时间线（9 月 Pilot · 10 月上线）。

## 四、录屏素材拍摄规格（发给素材团队）

1. **分辨率**：最高可用分辨率录制（4K 或 2 倍缩放），后期推镜头才不糊。
2. **干净的演示环境**：专用演示账号、干净测试数据；隐藏无关标签页、通知、书签栏、个人信息。
3. **每个场景单独录**，一条完整不间断；同场景多录 2–3 条备选。
4. **操作节奏放慢**：每次点击前后停 1–2 秒，给后期留剪辑缓冲；后期会加速，宁慢勿快。
5. **光标**：匀速直线移动，不要来回晃；如用 Screen Studio 类工具可后期平滑/放大光标。
6. **录窗口而非全屏**，保留 UI 完整边界，便于后期放进 3D 浮动窗口。
7. 每条素材命名：`场景编号-内容-take序号`（如 `S2-换问法-口语版-t2.mov`）。
8. 演示中出现的问答内容需业务确认为可对外展示的批准口径。

## 五、"大片感"后期手法清单（朴实录屏 → 成片）

- [ ] 录屏不全屏平铺：放进带阴影/景深的浮动窗口，背景用品牌色渐变或虚化场景。
- [ ] 推拉镜头（punch-in）：关键按钮、回复气泡出现时推近 20–40%。
- [ ] 速度变化：过程性操作 2–4 倍快进，关键瞬间（答案出现、转人工触发）恢复原速甚至放慢。
- [ ] 高亮聚焦：聚光/周边压暗或模糊无关区域。
- [ ] 动态字幕（kinetic typography）打关键词和数字，与音乐卡点。
- [ ] 音效设计：键盘、点击、消息提示、whoosh——音效贡献一半的大片感。
- [ ] 统一调色：录屏、动画、字幕用同一套品牌色板。
- [ ] 转场用"内容驱动"而非模板转场：上一镜的气泡飞出成为下一镜的标题等。
