# AI 客服项目宣传视频 · 脚本与分镜工作稿（v0.2）

> 状态：叙事方向已确认——「旧模式 vs 新模式」对比叙事；旁白为初稿待改。
> 时长目标：90 秒（可裁剪 60 秒 / 30 秒版）。
> 素材：真实 demo 录屏（产品部分）+ AI 生成（氛围与情绪镜头）。

## 一、定位（已确认）

| 问题 | 结论 |
|---|---|
| 给谁看？ | 管理层。**治理与指标另有正式汇报，本片不讲**；本片只回答一件事——**为什么做这件事** |
| 一句话核心信息 | 客户和客服天天来回打电话问后台；上线后 AI agent 直接解决——**省时间，体验好** |
| 依据 | 后台 calling 量下降（大数字动效，Pilot 后回填真实数据） |
| 看完记住什么 | 以前客户看着手机发愁；现在贷款办成，人已经提上车开在路上 |
| 收尾 | BMW logo + 品牌音效，帅气定版 |

## 二、90 秒叙事结构（五段）

### ① 旧模式 · 来回拉扯（0–15s）

- 客户侧影/背影看着手机发愁（AI 生成，不露正脸），屏幕微光打在脸上。字幕浮现：「贷款进度怎么样了？」
- 三方拉扯蒙太奇：客户打客服 → 客服放下这通、拿起另一通打后台 → 后台翻查 → 回电……画面在三个空间来回切，时间字幕跳动（周三 10:20 → 周四 15:00 → 周五……）。
- 音效：拨号音、占线嘟嘟声、翻纸声，越来越密。
- 定格黑屏：**「一个问题，N 通电话，XX 小时」**（数字待业务确认口径）。

### ② 转折（15–20s）

- 音乐骤停。一行字幕：「如果，一次就能问对地方？」
- AI agent 对话界面在黑场中亮起。

### ③ 新模式 · AI agent 直接解决（20–55s）—— 录屏主战场

| 场景 | 演示内容 | 传递的点 |
|---|---|---|
| 1 | 客户口语化提问贷款问题 → AI 秒回准确答案，推镜头到回复气泡 | 一次问对地方，秒级响应 |
| 2 | 进度/材料类问题 → AI 直接给出进度与办理指引；插入对比画面：后台的电话安静了 | 客服不再来回打后台 |
| 3（短，一拍带过） | 超范围问题顺畅转人工 | 体验不断档，兜底可靠 |

### ④ 依据 · 数据下降（55–70s）

- 大数字动效：**后台 calling 量**曲线下坠，数字从 X 滚动降到 Y；辅以「平均解决时长：XX 小时 → 分钟级」。
- 字幕：「不是感觉变快了，是数据降下来了。」
- ⚠️ 数据口径：Pilot 前用预估口径需明确标注"预期"；Pilot 后回填真实数据出正式版。

### ⑤ 升华 · 开在路上（70–90s）

- 呼应开场同一个"客户"：不再看着手机发愁 → 提车、上车（手部/剪影镜头）→ BMW 行驶在公路上，光线过肩、轮毂特写。
- 旁白落点：「把时间还给客户。」
- 定版：**BMW logo + 品牌音效**，黑场收尾。

## 四、素材策略：真实录屏 + AI 生成

**原则：产品演示必须用真实录屏，氛围和情绪镜头交给 AI 生成。** 不用 AI 生成任何像产品界面的画面；人物一律背影、手部、剪影，不生成可辨认人脸。

| 素材类型 | 来源 | 用在哪 |
|---|---|---|
| 产品演示 | 真实 demo 录屏 | 第③段三个场景（全片核心） |
| 情绪镜头 | AI 图生视频 | 第①段发愁客户、电话蒙太奇；第⑤段提车、上车 |
| 数据动效 | AE / 剪映图文成片 | 第④段 calling 量下坠 |
| 行驶收尾镜头 | **优先 BMW 官方素材库成片** | 第⑤段公路镜头 |
| 旁白 | TTS 配 animatic（豆包 / ElevenLabs），定稿后决定是否换人声 | 全片 |
| 音乐 | Suno 或版权库 | 全片 |

### AI 镜头清单（第①⑤段，逐条生成）

1. 夜晚室内，男性背影坐在沙发看手机，屏幕冷光打在侧脸轮廓，情绪低落，浅景深，电影感
2. 客服工位特写：手放下一部电话、拿起另一部，桌面便签杂乱，快节奏
3. 后台办公室：翻文件、看屏幕的手部特写，日光灯冷色调
4. 手部特写接过车钥匙，暖色调，浅景深，微笑剪影
5. （备选，官方素材优先）BMW 行驶在沿海/山路公路，黄昏逆光，过肩镜头 + 轮毂特写

### ⚠️ 两条风险提示

- **AI 生成 BMW 实车极易翻车**：车标、双肾格栅、车身比例会畸变，管理层一眼能看出来。行驶与提车的车辆镜头优先用品牌官方素材库；AI 只做不含车标特写的氛围镜头，或用官方剧照走图生视频。
- **BMW logo 与品牌音效（brand sound）有严格 CI 规范**，即使内部视频也建议提前过品牌/传播团队确认使用方式。

### AI 生成实操要点

1. 先定分镜再生成，逐镜头写 prompt（镜头运动、光线、情绪）。
2. 生成 3–5 秒短镜头拼节奏，不追长镜头。
3. 先用生图（Midjourney / 即梦）定 3–4 张风格帧，确认后再图生视频（可灵 / 即梦 / 海螺 / Runway），比纯文生视频稳定。
4. 国内使用优先可灵 / 即梦，中文 prompt 与合规更友好。

## 四b、录屏素材拍摄规格（发给素材团队）

1. **分辨率**：最高可用分辨率录制（4K 或 2 倍缩放），后期推镜头才不糊。
2. **干净的演示环境**：专用演示账号、干净测试数据；隐藏无关标签页、通知、书签栏、个人信息。
3. **每个场景单独录**，一条完整不间断；同场景多录 2–3 条备选。
4. **操作节奏放慢**：每次点击前后停 1–2 秒，给后期留剪辑缓冲；宁慢勿快。
5. **光标**：匀速直线移动；可用 Screen Studio 类工具后期平滑/放大光标。
6. **录窗口而非全屏**，保留 UI 完整边界，便于放进 3D 浮动窗口。
7. 素材命名：`场景编号-内容-take序号`（如 `S2-进度查询-t2.mov`）。
8. 演示问答内容需业务确认为可展示的批准口径。

## 五、"大片感"后期手法清单

- [ ] 录屏放进带阴影/景深的浮动窗口，背景品牌色渐变或虚化场景。
- [ ] 关键瞬间推镜头（punch-in）20–40%：答案出现、转人工触发。
- [ ] 速度变化：过程操作 2–4 倍快进，关键瞬间原速或放慢。
- [ ] 聚光/压暗聚焦关键区域。
- [ ] 动态字幕卡音乐节点。
- [ ] 音效设计：拨号音、消息提示、whoosh——音效贡献一半大片感。
- [ ] 全片统一调色：旧模式冷色调 → 新模式暖色调，色彩本身讲故事。
- [ ] 内容驱动转场：手机屏幕亮起转场到 AI 界面、气泡飞出成为数据卡片。
